# 🎥 Motion Detector to Telegram

Детектор движения в реальном времени через веб-камеру с автоматической записью видео и отправкой уведомлений в Telegram-бот.

## ✨ Возможности
- 📹 **Запись видео**: При обнаружении движения автоматически записывается ролик (длительность настраивается).
- 📩 **Уведомления в Telegram**: Мгновенная отправка текста и видеофайла в указанный чат.
- 🛡️ **Умная детекция**: Алгоритм на базе OpenCV с фильтрацией шума и настройкой чувствительности.
- 🌐 **Поддержка прокси**: Возможность работы через SOCKS/HTTP прокси для обхода блокировок Telegram API.
- ⚙️ **Гибкая настройка**: Порог срабатывания, кулдаун и длительность записи вынесены в конфиг.

## 📋 Требования
- Python 3.13+
- Веб-камера (встроенная или USB)
- Установленный [uv](https://github.com/astral-sh/uv) (менеджер пакетов)
- Установленны в PATH [ffmpeg](https://www.gyan.dev/ffmpeg/builds/)


## 🚀 Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/mayn3r/motion_detector_to_tg.git
cd motion_detector_to_tg
```

### 2. Установка зависимостей (uv)
```bash
uv pip install .
```

### 3. Конфигурация
Создайте файл .env в корне проекта  укажите свои данные:
```env
# Telegram Bot
BOT_TOKEN=123456:ABC-DEF...
CHAT_ID=987654321

# Proxy
PROXY_URL=socks5://user:pass@ip:port

# Настройки детектора
COOLDOWN=10          # Длительность записи видео в секундах
AREA_THRESHOLD=1000   # Минимальная площадь объекта для срабатывания (фильтр шума)
```

### 4. Запуск
```bash
uv run main.py
```

## ⚙️ Как настроить чувствительность
`AREA_THRESHOLD`: Если детектор реагирует на мух, блики или шум матрицы — увеличьте значение (например, до 1500). Если не видит человека — уменьшите (до 500). <br>
`COOLDOWN`: Время, которое камера будет писать видео после срабатывания.
